# AIL Semantic Layer — Спецификация v0.1

## 1. Назначение

Система предназначена для единого представления:

- Технического кода (AIL / Binary AST)
- Контента (карточки, статьи, сущности)
- Связей между ними
- Эволюции во времени (lineage)
- Визуализации и машинного обмена (M2M)

Код и контент рассматриваются как две стороны одного целого.

## 2. Основные слои

```text
Уровень 0:  Core (центр, ✕)
Уровень 1:  Гексагон доменов (GERMES)
Уровень 2:  Модули и группы (⬡)
Уровень 3:  Функции и Сущности
Уровень 4:  Binary AST / Детали
Уровень 5:  Исполнение + Ownership + VizGraph
```

## 3. Гексагон доменов (GERMES)

```text
1. Devices      --+   →
2. Payments     -+-   ◇
3. Users        +--   △
4. Logistics    -++   ▲
5. Content      +-+   □̸
6. Analytics    ++-   —
```

Центр = проекция пространственной диагонали куба (`---` ↔ `+++`).

## 4. Визуальный язык рёбер (Star Gate)

| Символ | Значение                     | Типичное применение              |
|--------|------------------------------|----------------------------------|
| `—`    | Обычная связь                | Control, базовая зависимость     |
| `→`    | Направленный поток           | Data-flow, порождение            |
| `△`    | Влияние                      | Воздействие одной сущности на другую |
| `◇`    | Устойчивая связь             | Связи между доменами             |
| `□̸`   | Shared                       | Разделяемый доступ               |
| `▲`    | Unique / изменяющее          | Mutable borrow, активное изменение |
| `✕`    | Конфликт                     | Взаимоисключение (Transfer ✕ Refund) |
| `⬡`    | Гиперребро / группа          | Atomic group, Module             |

## 5. Типы вершин (VertexKind)

- `Domain` — вершина гексагона
- `Module` — гиперребро-модуль
- `Function` — AIL-функция
- `Entity` — конкретная сущность (товар, пользователь, заказ)
- `Detail` — поле/деталь сущности
- `Article` — статья / обзор

## 6. Ownership (базовый)

Состояния:
- `Owned`
- `Borrowed` (shared) → `□̸`
- `BorrowedMut` (unique) → `▲`
- `Consumed`

Конфликт `✕` возникает при пересечении уникальных доступов.

## 7. Lineage

Каждый модуль и каждая важная сущность имеют:

- `content_hash`
- `parent_hash`
- `generation`
- `origin` (`Manual`, `Healed`, `Updated`…)
